Q3 — Perbedaan yang Dirasakan Sebelum dan Sesudah Redis

Pertanyaan
Apa bedanya yang bisa saya rasakan tanpa Redis sebelumnya?
Jawaban
TANPA REDIS (sebelumnya): - Setiap halaman dibuka = query ke SQL Server remote (ada latensi jaringan) - Session disimpan di file = lambat saat banyak user concurrent - Data yang sama di-query berulang kali setiap request - Jika SQL Server sibuk = semua user menunggu DENGAN REDIS (sesudah): - Data sering diakses tersimpan di RAM = response <1ms - Session di Redis = lebih cepat dan aman untuk multi-server - Query ke SQL Server hanya saat cache expired (misal tiap 5 menit) - Bisa handle 10.000+ concurrent session tanpa beban ke database CONTOH NYATA YANG TERASA: - Dashboard load statistik: dari ~800ms menjadi ~50ms - Halaman yang sama dibuka 100 user = hanya 1x query ke SQL Server - Login/logout lebih cepat (session di RAM bukan file) - Dropdown data master (dokter, kamar, poli) jauh lebih responsif

Q5 — Menangani SQL Server Windows dari Aplikasi Linux

Pertanyaan
Bagaimana menangani database yang menggunakan SQL Server tetapi aplikasi di Linux? Karena kalau mau linked server, install SQL Server di Linux itu tidak optimal sama sekali.
Jawaban
Solusi terbaik: KONEKSI REMOTE via TCP/IP (tidak perlu install SQL Server di Linux sama sekali) ARSITEKTUR: [Linux Server / Docker] ----port 1433 TCP----> [Windows Server SQL Server] KONFIGURASI .env DI LARAVEL: DB_CONNECTION=sqlsrv DB_HOST=192.168.1.100 (IP Windows Server SQL Server) DB_PORT=1433 DB_DATABASE=nama_database DB_USERNAME=sa DB_PASSWORD=password_anda KONFIGURASI DI WINDOWS SQL SERVER: 1. SQL Server Configuration Manager - Protocols for MSSQLSERVER - TCP/IP - Enable - IP Addresses - IPAll - TCP Port = 1433 2. Windows Firewall - Inbound Rules - New Rule - Port 1433 - Allow 3. Mode Autentikasi - Server Properties - Security - SQL Server and Windows Authentication mode PHP EXTENSIONS (sudah diinstall di Dockerfile via PECL): - sqlsrv (driver utama SQL Server) - pdo_sqlsrv (PDO driver untuk Laravel) Verifikasi extension: docker-compose exec app php -m | grep sqlsrv

Q6 — Apakah Ada Delay Aplikasi Linux ke Database Windows?

Pertanyaan
Apakah kalau begitu aplikasi di Linux dan database di Windows akan terjadi delay saat munculin data atau insert data? Atau tidak terasa bedanya?
Jawaban
JAWABANNYA: Tergantung di mana server ditempatkan. SKENARIO 1 - SATU JARINGAN LAN (kondisi ideal untuk rumah sakit): Latensi : 0.1ms - 1ms Dampak : TIDAK TERASA sama sekali oleh pengguna Keterangan: Server app dan SQL Server di gedung yang sama (ruang server) SKENARIO 2 - BEDA GEDUNG / WIFI: Latensi : 1ms - 10ms Dampak : Hampir tidak terasa, apalagi dengan Redis cache aktif SKENARIO 3 - VIA INTERNET / VPN: Latensi : 20ms - 100ms ke atas Dampak : Bisa terasa untuk query berat yang tidak di-cache SOLUSI UNTUK MEMINIMALKAN DELAY: 1. Redis Cache - Data yang sering dibaca di-cache di RAM - Query ke SQL Server hanya saat cache expired 2. Connection Pooling - Laravel handle otomatis, tidak connect ulang setiap request 3. Eager Loading - hindari N+1 query - Contoh: Patient::with(['dokter', 'kamar', 'diagnosa'])->get() 4. Database Index - Pastikan kolom WHERE sudah di-index: no_rm, tgl_masuk, id_dokter, id_poli KESIMPULAN: Dalam jaringan LAN rumah sakit, delay koneksi Linux (Docker) ke Windows (SQL Server) praktis tidak terasa.
